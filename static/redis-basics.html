Redis Basics

What is Redis

Redis is an open-source, in-memory data structure store, used as a database, cache, and message broker. It supports data structures such as strings, hashes, lists, sets, sorted sets with range queries, bitmaps, hyperloglogs, geospatial indexes, and streams.

Getting Started

  1. Open a terminal (WSL on windows).
  1. To install Redis, run the following command:
    sudo apt-get install redis
  1. Run this command to start the Redis server:
    redis-server
  1. To interact with Redis database, run the following command in another terminal while not terminating the previous one:
    redis-cli

Redis CLI

Basic Commands

The commands can be both upper and lower case.

  1. quit Exits from Redis CLI.
  1. SET key value Sets value to key.
  1. GET key Gets value of key.
  1. DEL key Deletes of key and its value.
  1. EXISTS key Checks if the key exists.
  1. KEYS * Gets list of all the keys.
  1. FLUSHALL Clear the database.

Note: To set values with more than one words, enclose the values in double quotes, like this:

SET key "value with more than one words"

Expirations Time

We can set the duration for which a specific key would exist called Time to Live (ttl). We can do this in two ways:

  1. Setting Expiration Time
    1. Setting key-value, then setting ttl . (time has a unit of seconds)
      SET key value
EXPIRE key time
    1. Setting ttl directly with setting key-value.
      SETEX key time value
  1. Seeing Expiration Time

    This returns the remaining time in seconds.

    ttl key

Redis Data Structures

Lists

By default, Redis saves and returns string values. For example, the SET and GET commands saves and returns string values.

To work with lists (arrays), the following are some basic commands:

  1. LPUSH list-name value Pushes value to the left (start) of the list.
  1. RPUSH list-name value Pushes value to the right (end) of the list.
  1. LPOP list-name Pops value from the left (start) of the list.
  1. RPOP list-namePops value to the right (end) of the list.
  1. lrnage list-name <start> <end> Returns the elements of the list form index <start> to <end> .

Multiple values can be added (But cannot remove multiple values) to a list in a single command as following:

# List
LPUSH list-name value1 value2 value3
RPUSH list-name value1 value2 value3

Sets

Sets are like list with difference being that it can store unique values only. Following are some commands for working with sets:

  1. SADD set-name value Adds value to the set.
  1. SREM set-name value Removes value from the set.
  1. SMEMBERS set-name Returns all the values of the set.

Multiple values can be added to a set and removed from a set in single command as following:

# Adding values to a set
SADD set-name value1 value2 value3

# Removing values from a set
SREM set-name value1 value2 value3

Hashes

Hashes are data structures in Redis that stores field-value pairs. Here are some basic operation of hashes:

  1. HSET hash-name field1 value1 field2 value2 field3 value3 Creates new fields within the hash.
  1. HGET hash-name field Returns the value of a specific field.
  1. HMSET hash-name field1 value1 field2 value2 field3 value3 Works similar to HSET .
  1. HMGET hash-name field1 field2 field3 Returns values of multiple fields
  1. HDEL hash-name field Deletes field
  1. HEXISTS hash-name field Checks if the field exists.
  1. HGETALL Returns all the field-value pairs.

Redis Caching in Python

Code Example

Here is an example code that demonstrates a simple Python application that utilizes Redis for caching results of a mathematical operation. 

import time

from redis import StrictRedis
from redis_cache import RedisCache

client = StrictRedis(host='localhost', port=6379, decode_responses=True)
cache = RedisCache(redis_client=client)


@cache.cache(ttl=60)
def math_operation(n):
    return sum(range(n))


def main():
    n = int(input("Enter value of n: "))
    t0 = time.time()
    result = math_operation(n)
    t1 = time.time()
    
    print(f"Result: {result}, Time: {(t1 - t0)*1000:.2f}ms")


if __name__ == "__main__":
    main()

Explanation

Here's a breakdown of how caching with Redis is implemented in Python, referencing the given code:

  1. Importing Required Libraries:
    • time: Used for calculating the execution time of the operation.
    • StrictRedis: A class from the redis package that provides a client for interacting with Redis.
    • RedisCache: A caching mechanism that interfaces with Redis, from the redis_cache package.
  1. Setting Up Redis Client:
    • A StrictRedis client is instantiated with connection details to the Redis server (localhost on port 6379). The decode_responses=True parameter ensures responses from Redis are decoded from bytes to strings.
  1. Creating a RedisCache Instance:
    • An instance of RedisCache is created, passing the Redis client as an argument. This instance is used to manage caching.
  1. Caching Decorator:
    • The @cache.cache(ttl=60) decorator is applied to the math_operation function. This decorator automatically caches the result of the function for 60 seconds (ttl=60). If the function is called again with the same arguments within this time frame, the cached result is returned instead of recalculating.
  1. Function to be Cached:
    • math_operation(n): A simple function that calculates the sum of numbers from 0 to n-1. This is the function whose result is cached.
  1. Main Function:
    • Prompts the user to enter a value for n.
    • Measures and prints the execution time of math_operation(n), demonstrating the potential time savings from using caching. The first call to math_operation with a specific n will compute and cache the result; subsequent calls with the same n within 60 seconds will fetch the result from the cache, likely reducing the execution time.
  1. Execution Entry Point:
    • The standard Python idiom to specify the entry point of the application.

Key Takeaways

  • Caching with Redis in Python can significantly improve the performance of applications by storing the results of expensive function calls and reusing them.
  • The redis and redis_cache libraries provide convenient interfaces for interacting with Redis, including decorators for caching function results.
  • The ttl parameter in the caching decorator specifies how long (in seconds) the cached result should be considered valid. After this period, the result is recomputed on the next function call.
  • This approach is particularly useful for applications with repeatable and time-consuming operations.
